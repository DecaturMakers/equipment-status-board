# Member Demo Checklist

Talking/demo points to hit when training members hands-on instead of using
the slide deck (`members.md`). Demo each live where you can.

- **The status colors** — 🟢 Operational (good) · 🟡 Degraded (works, use with care) · 🔴 Down (do not run). Status is live, driven by open reports.
- **Status dashboard (web)** — open the ESB address, no login; cards grouped by area, non-green show the issue + ETA.
- **Kiosk screens** — the wall displays auto-refresh; just look up before walking to a tool.
- **Scan a QR sticker** — phone camera → tap link (no app); show the big status indicator, issue, and Equipment Info/docs link.
- **Check Known Issues first** — if it's already listed, don't re-report.
- **Report a problem** — on the QR page: name + description (required), severity, safety-risk checkbox, optional photo → Submit; show the confirmation page.
- **What happens next** — record created, status updates everywhere, follow the named Slack channel for progress.
- **Slack from anywhere** — `/esb-status` (all / one area / one machine) and `/esb-report` (form popup).
- **The network caveat** — 📶 web + QR only work on WiFi; **Slack works anywhere**.
- **Where to get help** — the Help/Docs link → Members Guide; ask any staff member.
